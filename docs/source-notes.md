## Gistから教材化するときの方針

元のGistは、1ファイルでWebアプリの脆弱性を見せるための最小構成でした。このリポジトリでは、その意図を保ちながら、演習で使いやすいように次の整理をしています。

- `main.py` と掲示板HTMLを `app/` に分離した。
- Python 3系と現在のPeeweeで起動するよう、DB接続を `SqliteDatabase` にした。
- SQLインジェクション、XSS、CSRF、コマンドインジェクションは意図的に残した。
- 失敗時にアプリが落ちる箇所は、教材として観察しやすいエラー画面にした。
- XSS、CSRF、ダミーペイロード配布用のローカル補助サーバを追加した。

## 取り扱い

このリポジトリは、脆弱性の原理を理解し、対策を実装するための教材です。演習用ペイロードはローカル環境に閉じて使ってください。
